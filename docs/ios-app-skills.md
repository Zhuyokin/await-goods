# AwaitGoods iOS App Skills

更新时间：2026-05-03

## 1. 产品定位

AwaitGoods 是一款面向 iOS 17.0 及以上系统的本地优先心愿清单与购物决策应用，强调克制、安静、轻量的记录体验。核心目标不是促销或记账，而是帮助用户记录想买的物品、跟进存钱进度、降低冲动消费，并通过多主题、多语言和小组件提供持续可用的日常工具体验。

## 2. 核心能力

### 2.1 愿望记录与管理

- 新增、编辑、删除心愿条目
- 支持价格、已存金额、分类、链接、备注、优先级与状态
- 支持筛选、搜索、拖拽排序与批量整理

### 2.2 进度与外观

- 支持主列表与详情页显示存钱进度
- 支持桌面小组件展示摘要与进度
- 支持深色模式、浅色模式与跟随系统
- 支持多种主题风格切换，覆盖至少以下方向：
  - 日系清新极简风格
  - 奶油极简风格
  - Apple 极简风格
- 允许后续按 UI 精美程度继续迭代和替换主题方案

### 2.3 首次体验与品牌入口

- 提供新手引导页，覆盖记录、存钱进度、整理、小组件与备份等基础能力
- 提供 Launch 启动页，展示品牌图形与应用名称

### 2.4 数据与支持能力

- 支持离线 JSON 导入与导出
- 支持联系客服功能
- 支持 App 内查看技术支持与隐私政策

## 3. 多语言与合规范围

### 3.1 首批支持语言

- 简体中文
- 繁体中文
- English
- 日本語
- 한국어
- Deutsch
- Italiano
- Français

### 3.2 合规页面

- 技术支持页面需支持多语言
- 隐私政策页面需支持多语言
- App 内与 App Store Connect 中的相关链接应指向对应语言页面或具备明确语言回退策略

## 4. 技术要求

- 开发语言：Swift
- 目标平台：iOS
- 最低系统版本：iOS 17.0
- 需兼容深色模式、本地化、WidgetKit、小组件刷新、JSON 文件导入导出与系统分享/文件选择流程

## 5. 关于页面要求

关于页面是产品信息和用户自助入口的统一承载页，至少包含以下模块：

- App 介绍
- 当前版本信息
- 客服微信：`zhuyokin`
- 点击微信号后可复制到剪贴板，并显示 Toast 提示
- 主题设置入口，覆盖常用主题切换
- 多语言切换入口
- 离线数据导入入口
- 离线数据导出入口
- 技术支持页入口
- 隐私政策页入口

## 6. 工程文档规范

工程根目录需维护 `docs` 目录，并按语种拆分独立文档。每个语种必须包含：

- 技术支持文档
- 隐私政策文档
- ASO 文档
- App Store 截图文案

ASO 文档统一顺序如下：

1. 基础资讯
2. 应用描述
3. 此版本新增内容
4. 关键词

其中：

- 副标题不得超过 30 个字符
- 推广文案不得超过 170 个字符

## 7. 版本管理规范

工程根目录需维护 `versions` 目录，并按 `v1.0.1.md`、`v1.0.2.md` 这一类格式保存版本更新记录。

每次版本更新时，必须同步执行以下动作：

1. 新增或更新对应版本记录文件
2. 更新根目录 `readme.md`
3. 校对 `docs` 中各语种 ASO、支持、隐私与截图文案是否需要同步调整
4. 确保版本内容与实际功能、小组件、主题、多语言范围一致，防止上下文丢失

## 8. 建议维护方式

- 功能发生变化时，优先更新本文件中的能力说明，再同步具体语种文案
- 上架文案变化以 `docs` 为准，版本节奏以 `versions` 为准
- 若 `ios/version` 仍保留历史版本资料，应将其视为迁移来源，不应替代根级文档入口
