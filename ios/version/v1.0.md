# 候物 iOS v1.0 产品需求、原型与 UI 风格

## 1. 产品概述

### 1.1 产品名称

- 中文名：候物
- 英文名：AwaitGoods
- Slogan：等一等，再入手
- 产品寓意：等候入手、待购好物。把「想买」先安放下来，让真正值得买的东西自然留下。

### 1.2 产品定位

候物是一款基于 iOS 17 的克制极简心愿购物清单 App。它不鼓励冲动消费，也不做复杂的比价导购，而是帮助用户记录、冷静、筛选和回看想买的物品。

v1.0 重点是一个离线优先、轻量、好看、无负担的本地工具：快速记下想买的东西，为每个物品设置等待期、预算、优先级和购买状态。

交互气质保持中文工具 App 的低学习成本，同时视觉对齐日系极简：和纸白 / 淡灰米底、墨色文字、低对比浅卡片、细线分割、克制浅阴影、温和小圆角、线性图标和文字按钮。用户不需要理解复杂信息架构，打开后就能像写下一条生活备忘一样安放一个想买的东西。

### 1.3 目标用户

- 容易被种草，但希望减少冲动消费的人。
- 喜欢极简工具，不想被社区、推荐流、促销信息打扰的人。
- 需要记录待购物品、礼物灵感、长期观察商品的人。
- 注重隐私，希望购物心愿数据优先保存在本地的人。

### 1.4 核心价值

- 先候后买：给每个心愿物品一个自然冷静期。
- 轻记录：输入少、路径短、保存快。
- 可回看：看见自己真正持续想要的东西。
- 不打扰：无信息流、无广告、无复杂社交。

### 1.5 竞品与工具体验启发

参考竞品截图中值得吸收的部分：

- 极简购物清单：白底、黑字、细分割线，几乎没有装饰。
- 勾选与划线：完成态一眼可见，操作成本低。
- 编辑 / 完成：进入编辑态后集中处理删除、排序、修改。
- 键盘内联新增：输入时不打断当前列表上下文。
- 颜色标记：用低饱和浅色突出某个条目，而不是做复杂标签系统。
- 小组件：把最重要的待处理条目放到桌面和锁屏，提高回看频率。

候物的取舍：不照搬竞品的英文标题、粉色主操作和促销感表达，也不走厚重装饰和强促销视觉。整体像一本安静的和纸手帐：和纸白、淡灰米、墨色、鼠尾草绿、淡木色和少量蓝灰构成轻量秩序，不像电商 App，也不像营销型清单 App。

## 2. v1.0 范围

### 2.1 必做能力

- 新增、编辑、删除心愿物品。
- 物品支持名称、价格、链接、备注、分类、优先级、等待期、目标日期、状态。
- 首页展示待候物品列表，支持搜索、筛选和排序。
- 首页采用日系极简单列列表，背景加入低存在感抽象线条与纸感块面；左侧提供状态图标控件。
- 物品详情展示记录信息、等待进度和购买决策提示。
- 支持状态流转：想买、已买、不买、再想想。
- 支持编辑模式：批量删除、调整状态、整理顺序。
- 支持低饱和颜色标记，用于临时突出重要物品。
- 支持本地通知提醒：等待期结束、目标日期到达。
- 支持 WidgetKit 小组件：小号展示 1 个，中号最多展示 3 个正在想买的物品。
- 设置作为底部独立 Tab，不再从首页以弹窗进入。
- 支持本地数据持久化，默认离线可用。
- 支持浅色、深色模式。

### 2.2 暂不做能力

- 不做账号系统。
- 不做商品抓取、自动比价、第三方电商接入。
- 不做社区、分享广场和推荐流。
- 不做云同步，后续版本可考虑 iCloud。
- 不做复杂预算账本，仅保留轻量价格和预算观察。
- 不做复杂主题皮肤，仅提供默认浅色、深色和系统跟随。

### 2.3 iOS 17 技术方向

- UI：SwiftUI。
- 数据：SwiftData 本地存储。
- 状态管理：Observation。
- 通知：UserNotifications。
- 小组件：WidgetKit。
- 系统体验：支持 Dynamic Type、深色模式、触感反馈、系统分享打开链接。
- 可选扩展预留：App Shortcuts、iCloud 同步。

## 4. 数据模型

### 4.1 WishItem

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 本地唯一标识 |
| title | String | 物品名称，必填 |
| price | Decimal? | 预计价格 |
| link | URL? | 商品链接 |
| note | String? | 备注、种草理由、尺寸信息等 |
| category | String? | 分类，如数码、衣物、家居、书影音 |
| priority | Int | 优先级：1 低、2 中、3 高 |
| status | Enum | waiting、bought、released、paused |
| markColor | String? | 低饱和标记色，如 green、yellow、pink、gray |
| sortIndex | Int | 用户手动排序索引 |
| createdAt | Date | 创建时间 |
| updatedAt | Date | 更新时间 |
| waitUntil | Date? | 等待期结束时间 |
| targetDate | Date? | 目标购买日期 |
| notifyEnabled | Bool | 是否开启提醒 |

### 4.2 状态定义

- 想买：仍在等待和观察。
- 已买：已经购买。
- 不买：确认不买，保留记录。
- 再想想：暂时不决策，可重新回到想买。

状态在首页用轻量视觉表达：想买为心形图标，已买为对勾，不买为叉号，再想想为暂停图标。每个物品左侧都显示对应状态图标，用户应能在不进入详情的情况下理解当前状态。

## 5. 功能需求

### 5.1 首页

首页是打开 App 后「候物」Tab 的第一屏，展示用户当前正在等待的物品。页面气质为日系极简清单：和纸白淡底、抽象线条与纸感块面、单列浅卡、细线描边、极淡阴影、紧凑间距和低存在感背景，列表项可直接操作。

功能要求：

- 顶部显示「候物」和简短统计，右侧保留排序 / 整理菜单与新增按钮；设置入口放到底部独立 Tab。
- 搜索入口默认收起，可通过下拉或顶部搜索框进入，避免首屏显得复杂。
- 状态筛选支持：全部、想买、已买、不买、再想想，可放在搜索结果页或顶部轻量分段控件。
- 排序支持：最近添加、等待结束、价格高低、优先级；高级排序入口不占用首屏主要位置。
- 列表行展示左侧状态图标、物品名称、价格胶囊、等待剩余天数和轻量标记色。
- 列表行使用和纸白浅卡、细线描边和极淡阴影，间距保持紧凑。
- 已买物品显示对勾和普通文字；不买物品使用灰色文字和删除线。
- 空状态展示一句短文案：暂无候物。

交互要求：

- 点击物品文字区域进入详情。
- 点击左侧状态图标可在「想买」和「已买」之间快速切换；「不买」和「再想想」可一键回到「想买」。
- 左滑支持快速标记「不买」和删除，删除使用红色破坏性操作。
- 长按物品弹出快捷菜单：编辑、复制链接、标记颜色、标记已买、标记不买、再想想。
- 点击新增按钮进入快速新增；键盘弹出时仍保留当前列表上下文。
- 点击「编辑」进入编辑模式，支持多选删除、拖动排序；点击「完成」退出。

验收标准：

- 100 条本地物品列表滚动流畅，列表行高度稳定。
- 搜索和筛选可组合使用。
- 删除前需要二次确认。
- 编辑态、正常态、键盘输入态之间切换不丢失当前滚动位置。

### 5.2 新增 / 编辑物品

新增页用于快速记录心愿物品，输入负担要低。优先支持柔和奶油字段的即时记录体验：先记名称，其他信息稍后补。

字段要求：

- 物品名称：必填。
- 预计价格：选填，支持小数。
- 商品链接：选填，支持从剪贴板粘贴。
- 分类：选填，支持手动输入和最近使用分类。
- 优先级：低、中、高。
- 等待期：不设置、3 天、7 天、14 天、30 天、自定义。
- 目标日期：选填。
- 提醒：默认跟随设置。
- 备注：选填。
- 标记色：选填，默认无标记。

交互要求：

- 名称为空时禁用保存。
- 粘贴链接后不自动访问外部网络。
- 设置等待期后自动生成 waitUntil。
- 用户从首页点「＋」时，默认聚焦名称输入框并弹出键盘。
- 保存后列表中立即出现新条目，不强制进入详情。
- 保存成功后返回首页或详情页。

验收标准：

- 新增最短路径不超过 2 步：输入名称，点击保存。
- 快速新增时不要求填写价格、分类和等待期。
- 编辑后 updatedAt 自动更新。
- 开启提醒时，系统通知权限未开启需引导申请。

### 5.3 物品详情

详情页帮助用户做购买前的冷静判断。详情不是营销页，保持工具型信息密度：标题清楚、分组简短、操作集中，视觉保持和纸白底和浅卡。

展示内容：

- 顶部显示物品名称、状态和价格，右上角为「编辑」。
- 等待进度：已等待天数、剩余天数、结束日期。
- 核心信息：分类、优先级、目标日期、链接、备注。
- 决策提示：基于状态和等待期显示一条克制短句。
- 标记色：如有标记，仅作为详情顶部的一条浅色背景或左侧细线。

状态操作：

- 标记已买。
- 标记不买。
- 重新想买。
- 再想想。
- 编辑。

决策短句示例：

- 还想要，就让它再等一天。
- 等待结束了，看看它是否仍然值得。
- 不买也很好，清单因此更轻。
- 已经入手，记得回看它是否真的被使用。

验收标准：

- 不同状态展示不同主操作。
- 有链接时可一键跳转浏览器。
- 等待期结束后状态不自动改变，只改变提示和提醒。
- 详情页返回首页时，列表中的状态、划线和标记色即时更新。

### 5.4 设置

设置页作为底部独立 Tab，保持轻量，只放影响长期使用的选项。

功能要求：

- 默认等待期：无、3 天、7 天、14 天、30 天。
- 通知提醒：总开关。
- 外观模式：跟随系统、浅色、深色。
- 数据管理：导出 JSON、清空全部数据。
- 关于：App 名称、版本号、极简说明。
- 小组件：选择展示数量、展示状态和是否显示价格。

验收标准：

- 清空数据前必须二次确认。
- 关闭通知总开关后，不再创建新的本地通知。
- 导出的 JSON 能完整包含物品字段。

### 5.5 小组件

小组件用于高频回看，不承载复杂操作。风格参考桌面待办的高可读呈现，但视觉上延续日系极简的和纸白底、低饱和色、细线描边和柔和行卡。

功能要求：

- 支持小号和中号 Widget。
- 小号展示 1 个正在想买的物品。
- 中号最多展示 3 个正在想买的物品，超过 3 个只在 App 内列表继续显示。
- 每条展示勾选框、名称和等待剩余天数。
- 点击小组件进入 App 首页或对应物品详情。
- 小组件背景使用和纸白 / 淡灰米背景，不使用复杂壁纸和强装饰图。

验收标准：

- 主 App 修改物品状态后，小组件能在合理时间内刷新。
- 无想买物品时，小组件展示「暂无候物」。
- 深色模式下小组件文字和分割线清晰可读。

## 6. 用户流程

### 6.1 快速记录

```text
打开候物
→ 点击新增
→ 输入物品名称
→ 可选填写价格 / 链接 / 等待期
→ 保存
→ 回到首页看到新物品
```

### 6.2 冷静购买

```text
收到等待期结束提醒
→ 打开物品详情
→ 查看等待进度和备注
→ 选择已买 / 不买 / 继续想买
```

### 6.3 回看不买的物品

```text
首页切换到不买
→ 浏览曾经想买但最终决定不买的物品
→ 如重新产生需求，可标记为重新想买
```

## 8. UI 风格

### 8.1 设计关键词

- 日系极简
- 和纸白 / 淡灰米
- 墨色文字
- 低对比浅卡片
- 细线描边
- 紧凑留白
- 低饱和鼠尾草绿
- 抽象纸感背景

### 8.2 日系极简方向

| 维度 | 采用 |
| --- | --- |
| 底色 | 和纸白、淡灰米，整体清淡但不发黄 |
| 圆角 | 主卡 12pt 左右，字段和 chip 10 到 14pt |
| 阴影 | 极淡短阴影，只用于层级提示 |
| 色彩 | 和纸底 + 墨色 + 鼠尾草绿 + 蓝灰 + 淡木色 |
| 质感 | 干净、安静、轻量，不做厚重拟物 |
| 字体 | Regular / Medium / Semibold，无衬线，大标题清楚但克制 |
| 背景 | 首页使用低透明度抽象线条、纸感块面和细分割，不写实绘制花朵 |

### 8.4 字体系统

- 字体选择：中文优先系统苹方，视觉上以 PingFang SC Regular / Medium 为基准。
- 标题：24 到 34pt，Medium 到 Semibold，首页和新增页保持明确但克制。
- 导航标题：17pt，Medium，保持系统可预期性。
- 商品名称：16 到 17pt，Medium；不买状态允许划线和次级灰。
- 正文说明：13 到 15pt，Regular，行距略宽松但页面间距收紧。
- 辅助文字：12 到 13pt，Regular，使用暖灰降低存在感。
- 价格数字：14pt，Medium + 等宽数字，直接用文字呈现。

### 8.5 布局与排版

- 间距收紧：页面主 VStack 约 12 到 14pt，列表行间距 8pt，卡片内元素 8 到 12pt。
- 页面结构：底部双 Tab，分别为「候物」和「设置」。
- 卡片圆角：主卡 12pt 左右，输入框与控件 10 到 14pt，小按钮 10pt。
- 阴影：只用极淡短阴影，半径 2 到 3pt，Y 偏移 1pt。
- 首页背景：和纸白渐变 + 低透明度抽象线条与纸感块面；列表和输入栏保持可读优先。
- 弹窗 / 浮层：和纸浅底，无强模糊、无强投影。

### 8.6 组件规范

- 状态标签：将“想买 / 已买 / 不买 / 再想想”做成状态图标 + 文字 + 淡底。
- 商品列表卡：统一和纸白浅卡 + 12pt 圆角 + 细线描边 + 极淡阴影；左侧使用状态图标，价格使用轻色胶囊。
- 顶部导航：标题“候物”使用清楚无衬线大标题；右侧按钮为小圆角浅卡与绿色新增按钮。
- 搜索框：和纸浅卡背景，14pt 圆角，允许极淡阴影。
- 筛选 / chip：状态图标 + 文字 + 浅底；选中态用低饱和绿色填充 + 和纸白文字。
- 底部输入框：和纸浅卡 + 极淡描边 + 轻阴影；提交按钮使用小圆角上箭头。
- 状态图标：想买使用心形，已买使用对勾，不买使用叉号，再想想使用暂停图标。
- 设置项：设置作为独立 Tab；图标为 14 到 15pt 线性图标，设置行使用字段底色 + 14pt 圆角。
- 小组件：和纸白 / 淡灰米背景，列表行用 14pt 圆角浅卡 + 极淡描边。

### 8.7 细节质感与交互

- 抽象纸感背景：使用低透明度线条、块面和细微纸感；不写实绘制花朵，不能覆盖主要文字和操作。
- 点击反馈：淡色高亮即可，无放大、无兴奋感动画。
- 列表滑动：保持系统滚动，避免夸张弹性。
- 状态切换：短淡入淡出，配合轻触感反馈。
- 删除确认：系统确认弹窗，破坏性文字使用柔红。

### 8.8 图标方向

App 图标建议以「候」或「物」的抽象组合为核心：

- 背景：和纸白、淡灰米或浅鼠尾草绿。
- 主体：简化的「候」字笔画、状态图标、抽象等待符号或纸感几何。
- 气质：像一本安静干净的和纸购物手帐，不像电商 App。
- 避免：购物车、火焰、折扣券、金币、强促销符号。

## 10. 通知策略

### 10.1 通知触发

- 等待期结束时提醒。
- 目标日期当天上午 9 点提醒。
- 同一个物品同一天最多提醒 1 次。

### 10.2 通知文案

- 标题：候物
- 内容：你等的「{物品名称}」可以再看看了。

### 10.3 权限策略

- 首次开启提醒时申请权限。
- 用户拒绝后，不反复弹窗。
- 设置页展示通知状态，并提供跳转系统设置的入口。

## 11. 非功能需求

- 启动速度：冷启动首屏目标 1 秒内可见。
- 隐私：v1.0 不上传任何数据。
- 稳定性：本地数据读写失败时给出明确提示。
- 可访问性：支持 VoiceOver、Dynamic Type、Reduce Motion。
- 国际化预留：数据字段不绑定中文展示文案。
