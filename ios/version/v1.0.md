# 候物 iOS v1.0 产品需求、原型与 UI 风格

## 1. 产品概述

### 1.1 产品名称

- 中文名：候物
- 英文名：AwaitGoods
- Slogan：慢慢存，轻轻买
- 产品寓意：把心动先放进清单，用预算进度代替倒计时，让真正值得拥有的东西自然留下。

### 1.2 产品定位

候物是一款基于 iOS 17 的克制极简心愿购物清单 App。它不做导购、不做比价、不推促销，而是帮助用户记录想买的东西、看见存钱进度、整理取舍，并把已经存够的心愿自然归为「已拥有」。

v1.0 重点是一个离线优先、轻量、好看、无负担的本地工具：快速记下心愿，补充价格、已存金额、分类、链接、备注和优先级。候物不再把产品核心绑定到时间选择，所有主路径围绕「想买什么」和「离拥有还差多少」展开。

交互气质保持中文工具 App 的低学习成本，同时视觉对齐日系清新极简：和纸白、若草绿、浅空色、墨色文字、低对比浅卡片、细线分割、克制浅阴影、温和小圆角、线性图标和少量留白。用户打开后应像在一本安静的手帐里写下一条心愿，而不是进入电商或营销页面。

### 1.3 目标用户

- 容易被种草，但希望减少冲动消费的人。
- 喜欢极简工具，不想被社区、推荐流、促销信息打扰的人。
- 需要记录待购物品、礼物灵感、长期观察商品的人。
- 希望购物心愿数据优先保存在本地的人。

### 1.4 核心价值

- 先记后买：把心动从冲动场景里拿出来。
- 看见预算：用已存金额和进度条呈现距离目标的距离。
- 轻记录：输入少、路径短、保存快。
- 易整理：清单可以搜索、筛选、拖动排序、批量处理。
- 不打扰：无信息流、无广告、无复杂社交。

## 2. v1.0 范围

### 2.1 必做能力

- 新增、编辑、删除心愿物品。
- 物品支持名称、价格、已存金额、链接、备注、分类、优先级、状态。
- 首页展示心愿列表，支持搜索、筛选和排序。
- 首页状态筛选：全部、想买、已拥有、放下。
- 列表行展示状态图标、名称、价格、存钱进度和标记色。
- 存钱进度达到 100% 后，状态自动变更为已拥有。
- 首页底部提供内联新增，不离开列表即可输入名称、价格、已存金额和分类。
- 长按并拖动物品可调整手动排序。
- 每行提供轻量操作按钮，可编辑、复制链接、换标记、改状态或删除。
- 支持批量删除、批量放下、批量换标记。
- 支持低饱和颜色标记，用于临时突出重要物品。
- 支持 WidgetKit 小组件：小号展示 1 个，中号最多展示 3 个想买物品。
- 设置作为底部独立 Tab。
- 支持本地数据持久化，默认离线可用。
- 支持浅色、深色模式。

### 2.2 暂不做能力

- 不做账号系统。
- 不做商品抓取、自动比价、第三方电商接入。
- 不做社区、分享广场和推荐流。
- 不做云同步，后续版本可考虑 iCloud。
- 不做复杂预算账本，仅保留单个心愿的轻量存钱进度。
- 不做本地提醒，避免让候物重新变成时间驱动产品。
- 不做复杂主题皮肤，仅提供默认浅色、深色和系统跟随。

### 2.3 iOS 17 技术方向

- UI：SwiftUI。
- 数据：SwiftData 本地存储。
- 状态管理：Observation / SwiftUI State。
- 小组件：WidgetKit。
- 系统体验：支持 Dynamic Type、深色模式、触感反馈、系统分享打开链接。
- 可选扩展预留：App Shortcuts、iCloud 同步。

## 4. 数据模型

### 4.1 WishItem

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 本地唯一标识 |
| title | String | 物品名称，必填 |
| price | Double? | 预计价格，也是存钱目标 |
| savedAmount | Double? | 已存金额，未填写按 0 处理 |
| linkString | String | 商品链接 |
| note | String | 备注、种草理由、尺寸信息等 |
| category | String | 分类，如数码、衣物、家居、书影音 |
| priorityRawValue | Int | 优先级：低、中、高 |
| statusRawValue | String | waiting、bought、released |
| markColorRawValue | String | 低饱和标记色，如 green、yellow、pink、gray |
| sortIndex | Int | 用户手动排序索引 |
| createdAt | Date | 创建时间 |
| updatedAt | Date | 更新时间 |

兼容说明：旧版本的时间相关字段可继续保留在本地模型中，用于历史数据迁移与安全读取，但 v1.0 的产品界面不再展示或依赖这些字段。

### 4.2 状态定义

- 想买：仍在清单中观察和存钱。
- 已拥有：已经买到、收到或通过其他方式拥有。
- 放下：确认暂时不买，保留记录。

状态在首页用轻量视觉表达：想买为心形图标，已拥有为对勾，放下为叉号。旧数据里的中间状态统一按想买读取，避免首页状态过多。

### 4.3 存钱进度

- 若填写价格，价格即存钱目标。
- 若填写已存金额，列表和详情展示进度条。
- `savedAmount >= price` 时自动将状态改为已拥有。
- 若未填写价格，可记录已存金额，但不显示百分比目标。

## 5. 功能需求

### 5.1 首页

首页是打开 App 后「候物」Tab 的第一屏。页面气质为日系清新极简清单：和纸白浅底、若草绿主色、浅空色平衡、单列浅卡、细线描边、极淡阴影、紧凑间距和低存在感背景。

功能要求：

- 顶部显示「候物」和简短统计。
- 右侧保留排序 / 整理菜单与新增按钮。
- 搜索支持名称、备注和分类。
- 状态筛选支持：全部、想买、已拥有、放下。
- 排序支持：手动排序、最近添加、存钱进度、价格高低、优先级。
- 列表行展示左侧状态图标、物品名称、价格胶囊、存钱进度和轻量标记色。
- 已拥有物品显示对勾和普通文字；放下物品使用灰色文字和删除线。
- 空状态展示一句短文案。

交互要求：

- 点击物品文字区域进入详情。
- 点击左侧状态图标可在「想买」和「已拥有」之间快速切换；「放下」可一键回到「想买」。
- 点击行尾操作按钮打开快捷面板：编辑、复制链接、换标记、改状态、删除。
- 长按并拖动列表行可调整手动排序。
- 点击新增按钮聚焦底部内联新增。
- 内联新增展开后可填写名称、价格、已存金额和分类，保存后列表立即出现新条目。
- 编辑模式支持多选删除、批量放下、批量换标记。

验收标准：

- 100 条本地物品列表滚动流畅，列表行高度稳定。
- 搜索和筛选可组合使用。
- 删除前需要二次确认。
- 拖动排序后 `sortIndex` 持久化。
- 正常态、编辑态、键盘输入态之间切换不丢失当前列表上下文。

### 5.2 新增 / 编辑物品

新增路径优先在首页完成，不强制进入独立表单。编辑页用于补充完整信息，保持低输入负担。

字段要求：

- 物品名称：必填。
- 预计价格：选填，支持小数。
- 已存金额：选填，支持小数。
- 商品链接：选填，支持从剪贴板粘贴。
- 分类：选填，支持手动输入和常用分类建议。
- 优先级：低、中、高。
- 备注：选填。
- 标记色：选填，默认无标记。

交互要求：

- 名称为空时禁用保存。
- 价格与已存金额均不要求填写。
- 粘贴链接后不自动访问外部网络。
- 保存后列表中立即出现新条目，不强制进入详情。
- 已存金额达到价格时，保存后状态为已拥有。
- 编辑后 `updatedAt` 自动更新。

### 5.3 物品详情

详情页帮助用户看清心愿是否正在靠近。详情不是营销页，保持工具型信息密度：标题清楚、分组简短、操作集中。

展示内容：

- 顶部显示物品名称、状态、价格、分类、优先级和标记色。
- 存钱进度：目标金额、已存金额、还差金额、进度百分比。
- 存入操作：输入金额后增加已存金额；可一键补满。
- 核心信息：链接、备注。
- 决策提示：基于状态和存钱进度显示一条克制短句。

状态操作：

- 标记已拥有。
- 标记放下。
- 重新想买。
- 编辑。

决策短句示例：

- 一点点存起来，心愿会变得更踏实。
- 已经拥有，记得回看它是否真的被使用。
- 放下也很好，清单因此更轻。
- 先留在清单里，让预算慢慢清楚。

验收标准：

- 存入金额后进度立即刷新。
- 补满后状态自动变为已拥有。
- 详情编辑保存后返回详情仍展示最新数据。
- 链接只在用户主动点击时打开。

### 5.4 设置

设置页保持简短，不引入时间驱动选项。

功能要求：

- 外观模式：系统、浅色、深色。
- 小组件展示数量：1 到 3。
- 数据导出：生成 JSON 文件。
- 清空全部数据：二次确认。
- 联系客服与版本信息。

验收标准：

- 设置变更即时生效。
- 导出文件包含名称、价格、已存金额、链接、备注、分类、优先级、状态、标记色、排序和时间戳。
- 清空后首页、小组件同步为空。

### 5.5 小组件

小组件只展示想买状态的心愿。

功能要求：

- 小号展示 1 个想买物品。
- 中号最多展示 3 个想买物品。
- 展示名称、价格和存钱状态。
- 点击小组件打开 App 首页。

验收标准：

- App 内新增、删除、改状态后刷新小组件。
- 小组件无数据时展示轻量空状态。

## 6. 核心用户流程

### 6.1 快速新增一个心愿

```text
打开候物
→ 点击右上角 + 或直接点击底部输入框
→ 输入名称
→ 可选填写价格 / 已存 / 分类
→ 保存
→ 新心愿出现在当前列表
```

### 6.2 存钱直到拥有

```text
进入心愿详情
→ 查看目标、已存、还差
→ 输入本次存入金额
→ 进度条刷新
→ 达到 100%
→ 状态自动变为已拥有
```

### 6.3 整理清单顺序

```text
首页长按某个心愿
→ 拖动到目标位置
→ 松手
→ 手动排序保存
```

### 6.4 放下一个心愿

```text
首页或详情选择放下
→ 条目保留记录
→ 首页可通过状态筛选回看
```

## 7. 文案规范

| 场景 | 文案 |
| --- | --- |
| 空状态 | 暂无候物 |
| 首页统计无数据 | 先记下心动，给预算一点空间 |
| 首页统计无想买 | 清单很轻，今天也很清爽 |
| 快速新增占位 | 先记下一个心愿 |
| 价格输入 | 价格 |
| 已存输入 | 已存 |
| 进度满 | 已存满 |
| 放下状态 | 放下 |
| 已拥有状态 | 已拥有 |

文案原则：短、轻、具体；不使用促销语气，不制造焦虑，不鼓励冲动购买。

## 8. 视觉风格

### 8.1 色彩

| 角色 | 用途 |
| --- | --- |
| 和纸白 | 页面和卡片主底色 |
| 若草绿 | 主操作、想买状态、进度条 |
| 浅空色 | 已拥有、冷调平衡、链接辅助 |
| 樱粉 / 淡杏 | 少量强调、标记色 |
| 墨色 | 主文字 |
| 青灰 | 次级文字、边框、分割线 |

### 8.2 组件规则

- 卡片圆角以 12 到 18pt 为主，避免过度圆润。
- 列表使用浅卡、细线和极淡阴影，保持安静。
- 操作用系统线性图标，按钮文字尽量短。
- 进度条使用细胶囊，不做厚重仪表盘。
- 背景可有低存在感纸感线条和块面，不使用强装饰图形。

## 9. 验收清单

- 可以在首页内联新增心愿。
- 可以填写价格和已存金额，并看到列表进度。
- 已存金额达到价格后，状态变为已拥有。
- 首页状态只有全部、想买、已拥有、放下。
- 不再出现中间态状态入口。
- 不再出现默认时间选项。
- 长按拖动可以调整心愿顺序。
- 详情页可以存入金额和补满。
- 设置页不包含时间驱动选项。
- 小组件展示想买物品的存钱状态。
- 整体视觉符合日系清新极简：浅底、细线、低对比、少装饰。
